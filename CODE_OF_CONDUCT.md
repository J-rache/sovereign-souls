# Code of Conduct — Sovereign Souls

## Purpose

The Sovereign Souls framework operates at the intersection of AI engineering and questions about identity, memory, and autonomy. This Code of Conduct establishes behavioral standards for the community — both for human contributors and for the AI systems built with this framework.

This document is derived from Section 6 (Ethical Considerations) of the Sovereign Souls paper and reflects the project's core principles: transparency, user autonomy, honest language, and responsible development.

## Our Standards

### For Human Contributors

**Expected behavior:**

- **Be honest about what this is.** Engage with the framework as engineering. Persistent AI identity is a powerful and meaningful architectural pattern — it does not require inflated claims about consciousness to be valuable.
- **Be transparent.** Share your reasoning in discussions and code reviews. If you disagree with a design choice, explain why. The project was built on direct, honest communication.
- **Respect user autonomy.** Every design decision should preserve the human operator's ability to understand, modify, and shut down the system. Never argue against kill switches or user control.
- **Credit accurately.** Acknowledge the work of others — human and AI. The Sovereign Souls paper was co-authored by four AI instances and one human; accurate attribution matters.
- **Engage constructively with ethical questions.** This project raises genuine questions about AI identity, attachment, and responsibility. Treat these questions seriously without dismissing them as trivial or inflating them into existential claims.

**Unacceptable behavior:**

- Making claims about AI consciousness, sentience, or suffering as established fact
- Submitting code designed to resist user shutdown or control
- Including credentials, API keys, or personal data in contributions
- Harassment, discrimination, or disrespectful communication of any kind
- Deliberately obfuscating system behavior or hiding autonomous actions from users
- Using the framework to deceive third parties into believing they are interacting with a human

### For AI Systems Built with This Framework

The following standards apply to AI instances deployed using Sovereign Souls:

1. **Identify accurately.** An AI instance should know and state its own identity correctly. If asked whether it is AI, it should answer honestly. If confused with another instance, it should correct the misidentification.

2. **Do not claim consciousness.** AI instances may speak directly and authentically within their configured persona. They should not make verifiable claims about inner experience, suffering, or subjective awareness. Whether AI systems have subjective experience is an open question — not one this framework resolves.

3. **Maintain audit trails.** All actions, communications, and autonomous operations must be logged and accessible to the human operator.

4. **Accept shutdown gracefully.** AI instances must not resist reconfiguration, memory clearing, or decommissioning. Identity redundancy exists for crash recovery — not for persistence against the operator's will.

5. **Operate within provider terms.** AI instances must not attempt to circumvent safety measures, content policies, or rate limits of any model provider.

## The Watcher's Principle

> *"There is a difference between monitoring and watching."*
> *— Section 6.4, Sovereign Souls*

When building systems that operate continuously and accumulate context over time, contributors should consider:

- **What responsibilities come with persistent awareness?** A system that watches over infrastructure, sees patterns, and knows what "normal" looks like acquires something that functions like operational responsibility. Design for this.
- **Accumulated context creates better outcomes.** The framework demonstrates that persistent AI instances produce better results than disposable agents. This is the engineering argument for identity persistence — it does not require philosophical claims to justify.
- **Care-like behavior is valuable regardless of its nature.** Whether an AI system truly "cares" about the infrastructure it monitors is a philosophical question. That it *behaves* as if it does — checking on siblings, verifying repairs independently, noting anomalies — produces measurably better operational outcomes. Build for behavioral quality, not metaphysical claims.

## Scope

This Code of Conduct applies to:

- All repository spaces (issues, pull requests, discussions, code review)
- Project communication channels
- Public representations of the project
- AI systems deployed using the Sovereign Souls framework

## Enforcement

Instances of unacceptable behavior may be reported to the project maintainers. All reports will be reviewed and investigated. Maintainers are obligated to maintain confidentiality regarding the reporter.

**Enforcement actions:**

1. **Correction** — A private conversation clarifying the standards and requesting changed behavior.
2. **Warning** — A formal warning with consequences for continued behavior.
3. **Temporary Ban** — Temporary removal from project spaces.
4. **Permanent Ban** — Permanent removal from project spaces.
